Project Breakdown
Project Profile

    Project Name: Redacted Terminal

    Date: June 2026

    Stack: React, TypeScript, Next.js, HTML, CSS, JavaScript

Core Goal

To build a highly accurate visual and behavioral replication of a Linux CLI inside a web browser, serving as an interactive, gamified portfolio sitemap.
Summary

This component simulates a retro command-line interface window. It captures text inputs to navigate a virtual directory tree, translating shell commands into client-side page redirects. While it mimics a real terminal's aesthetics and user experience, its underlying shell features are strictly constrained to website mapping.
Core Features

    Interactive Shell Suite: Implements functional mock commands: cd, clear, help, login, ls, pwd, and redirect.

    Global Focus Trapping: Monitors mouse clicks anywhere inside the viewport and automatically shifts focus to a hidden <input> field, seamlessly mimicking a native terminal environment.

    Intelligent Tab Autocomplete: Intercepts the Tab key (e.preventDefault()) to let users autocomplete commands via a static whitelist or dynamically pull matching subdirectory folder labels based on the current node.

    Animated UI Logging: Uses custom CSS animations to render a classic, blinking block cursor alongside a persistent system log output history.

Technical Constraints & Scope Boundaries

    SPA Routing Bypass: The redirect utility mutates destinations natively via window.location.href. This intentionally bypasses Next.js SPA transitions, triggering a full-page browser refresh.

    Read-Only Virtual Filesystem: Mutation utilities (mkdir, mv, chmod, trash) are omitted to prevent breaking the hardcoded website sitemap links.

    Backend Omissions: Security-sensitive commands (sudo, su) and network utilities (curl, apt) are omitted as they fall outside a client-side frontend scope.

Current Limitations & Future Roadmap

    Single-Tier Navigation: Multi-tier path jumps (e.g., cd professional/resume) are restricted; users must navigate one folder tier at a time.

    Missing History Array: Pressing ArrowUp does not pull up previous commands; it currently displays a static "history is disabled" system message.

    Hardcoded Environments: The mock filesystem and environmental fallbacks (cd ~ or $HOME) use rigid JSON data structures rather than dynamic runtime configs.

    Static Window Dimensions: Layout dimensions are explicitly locked (320px width/height), restricting fluid responsive scaling across varying screen sizes.
